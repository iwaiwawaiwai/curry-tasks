ボイスレコーダーの音声ファイルを文字起こしして、brain/00_Inbox/ に保存してください。

引数: $ARGUMENTS
（例: `会議名=熊野町役場MTG 参加者=岩政,森/熊野町2名 開始=13:00 終了=14:30`）

## 実行フロー

### 1. 引数パース
- 引数が省略された場合、USBの全WAVファイルを対象にする
- `開始` / `終了` が指定されていればその時間帯のファイルのみ対象にする

### 2. USBからローカルにコピー
- USB: `/Volumes/NO NAME/RECORD/` の WAV ファイルを確認
- ローカル保存先: `~/VSCode/tools/whisperx/audio/`
- 既にコピー済みのファイルはスキップ
- USBが接続されていない場合はローカルの既存ファイルを使う

### 3. WhisperX で文字起こし
- 実行環境: `~/VSCode/tools/whisperx/.venv/bin/python`
- スクリプト: `~/VSCode/tools/whisperx/transcribe.py`
- モデル: large-v2 / CPU / 日本語
- 出力: `brain/00_Inbox/_transcribe_raw.txt`（中間ファイル）

### 4. 議事録・メモに整形
引数の `会議名` / `参加者` を元に以下を判断：

**会議録フォーマット**（会議名・参加者が指定されている場合）
```markdown
# 議事録 - {会議名} ({日付})

## 基本情報
- 日時: {日付} {開始}〜{終了}
- 場所: （文字起こしから推測）
- 参加者: {参加者}

## 議題・課題
{文字起こしから抽出}

## 発言サマリー
{話者ごとに整理}

## アクションアイテム
{文字起こしから抽出}

## 補足・メモ
{その他の重要な発言}
```

**振り返りメモフォーマット**（「振り返り」「マック」等が含まれる場合）
```markdown
# 振り返りメモ - {日付}

## 内容
{文字起こしから整理}
```

### 5. brain/00_Inbox/ に保存
- フォーマット: `YYYY-MM-DD_{会議名またはtranscribe}.md`
- 中間ファイル（`_transcribe_*.txt`, `_transcribe_*.json`）はそのまま残す
- 確認なしで即保存

## 注意事項
- 文字起こしは時間がかかる（2.5時間の音声で30〜90分）
- バックグラウンド実行し、完了後に整形する
- USBは抜いても問題ないようローカルコピーを先に行う
