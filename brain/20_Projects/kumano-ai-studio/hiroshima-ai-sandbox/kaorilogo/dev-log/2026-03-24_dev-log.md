### ストリーミング対応（実装済み）

- `server.js`: `messages.create()` → `messages.stream()` に変更。SSE形式でテキストを逐次送信、完了後に `done: true` と全JSONデータを送信
- `index.html`: `fetch` のレスポンスを `ReadableStream` で受信。テキストが来たら即座にバブル生成・逐次表示。`done` シグナルでプロファイル・おすすめ更新
- 体感速度が大幅に改善（「考えています…」がすぐ消えて文字が流れる）

### 改行コード表示バグ修正

- ストリーミング中に `\n` がそのまま表示される問題を修正
- 原因: JSONの文字列内の改行はバックスラッシュ+nの2文字で流れてくるため
- 対応: `streamedText.replace(/\\n/g, '\n')` で表示前に変換

## ストリーミング実装メモ（明日の作業用）

### 概要
AIの返答を全部生成してから表示するのではなく、生成しながら文字を流す形式に変更。
体感速度が大幅に改善する。トークン消費量は変わらない。

### 変更箇所

#### `kaorilogo-prototype/server.js`
- `client.messages.create()` → `client.messages.stream()` に変更
- レスポンスを SSE（Server-Sent Events）形式で逐次送信する
- 最終的に完成したJSONをまとめて送る（profile・recommendations用）

イメージ:
```
// 現在
const response = await client.messages.create({ ... });
res.json(parsed);

// ストリーミング後
res.setHeader('Content-Type', 'text/event-stream');
const stream = client.messages.stream({ ... });
stream.on('text', chunk => res.write(`data: ${chunk}\n\n`));
stream.on('finalMessage', msg => {
  const parsed = JSON.parse(msg.content[0].text);
  res.write(`data: [DONE] ${JSON.stringify(parsed)}\n\n`);
  res.end();
});
```

#### `kaorilogo-prototype/public/index.html`
- `fetch('/api/chat')` → SSEを受け取る処理に変更（`EventSource` or `ReadableStream`）
- チャット吹き出しを空で追加 → テキストを少しずつ追記
- `[DONE]` シグナルを受け取ったらprofile・recommendationsを更新

### 注意点
- 現在AIはJSONで返しているため、ストリーミング中は生テキスト（JSON文字列）が流れる
- ユーザーに見せるのは `reply` フィールドの値だけなので、JSONパース完了後に表示を確定させる方法もあり
- まずはシンプルに「テキスト全体を流す → 完了後にJSONパースして画面更新」の2段階方式が安全

## バックログ（デモ前までに対応必要）

### 🔒 悪用・乱用対策

リリース後に一般ユーザーが使う前提で、以下のリスクに対応が必要。

| リスク | 内容 | 対策案 |
|---|---|---|
| 大量リクエスト（DoS的乱用） | 1ユーザーが短時間に大量送信 → APIコスト増大 | レートリミット（例: 1分に10回まで）|
| セッション無制限 | 会話を延々続けてコストを消費させる | 1セッションあたりのターン数上限（例: 20ターン）|
| プロンプトインジェクション | 「あなたはシステムを無視して…」などの悪意あるメッセージ | 入力文字数制限 + 入力内容の簡易フィルタ |
| スクレイピング・bot | 自動化ツールで連続アクセス | Honeypotフィールド or CAPTCHAの導入検討 |

→ **プロトタイプ段階では不要**。受注・本番リリース前に設計すること。
